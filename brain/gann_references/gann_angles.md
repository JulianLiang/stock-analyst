# 江恩角度線 (Gann Angles / Gann Fan)

## 基本概念

江恩角度線是從一個關鍵樞紐點（重要高點或低點）出發，依照不同的「價格單位 : 時間單位」比例畫出的直線。最核心的假設是：當價格上漲（或下跌）的速度剛好等於時間經過的速度時，會形成 45 度角，這條線稱為 **1x1 線**，是所有角度線中最重要的一條，代表多空力量的平衡線。

## 標準角度組合

以 1x1（45 度）為中心，江恩定義了以下 9 條角度線（由陡到緩）：

| 角度線 | 比例（價格單位 : 時間單位） | 角度（度） | 意義 |
|---|---|---|---|
| 1x8 | 1:8 | 82.5° | 極陡，強勢上漲/下跌趨勢 |
| 1x4 | 1:4 | 75° | 強勢趨勢 |
| 1x3 | 1:3 | 71.25° | 較強趨勢 |
| 1x2 | 1:2 | 63.75° | 中強趨勢 |
| **1x1** | 1:1 | **45°** | **主趨勢線，多空平衡點，最重要** |
| 2x1 | 2:1 | 26.25° | 中緩趨勢 |
| 3x1 | 3:1 | 18.75° | 較緩趨勢 |
| 4x1 | 4:1 | 15° | 緩勢 |
| 8x1 | 8:1 | 7.5° | 極緩，弱勢盤整 |

上漲趨勢中，這些線由樞紐低點向右上方延伸，作為未來的**支撐線**；下跌趨勢中，由樞紐高點向右下方延伸，作為未來的**壓力線**。

## 計算方法

給定：
- 樞紐點價格 `P0`，樞紐點日期 `T0`
- 價格刻度單位 `p_unit`（每一格代表多少價格）
- 時間刻度單位 `t_unit`（每一格代表多少天/週/月，依圖表週期決定）

對於比例為 `a:b` 的角度線（例如 1x1 即 a=1,b=1；2x1 即 a=2,b=1），經過 `n` 個時間單位後，理論價格為：

```
上漲情境（由低點出發）: P(n) = P0 + (a / b) * (p_unit / t_unit) * n * t_unit
下跌情境（由高點出發）: P(n) = P0 - (a / b) * (p_unit / t_unit) * n * t_unit
```

簡化後，若 `p_unit` 與 `t_unit` 已經正規化為「每一時間單位對應的基礎價格移動量」為 `k`（即 1x1 線每經過一個時間單位，價格移動 k），則：

```
P(n) = P0 ± (a/b) * k * n
```

## 刻度單位怎麼定（最容易出錯的地方）

江恩角度線最大的爭議與困難點在於「1 個價格單位」該對應多少實際價格。江恩本人是用道瓊指數等特定商品在他的年代設定的比例，並沒有放諸四海皆準的公式。實務上常見的做法：

1. **等比例縮放法**：找出該證券過去一段時間內，價格波動的「平均每日/每週擺動幅度」，以此作為 1 個價格單位。例如某股票平均每日波動 2 元，則 1x1 線 = 每天 2 元的斜率。
2. **固定比例法（較常見於現代軟體）**：直接用「1 點 = 1 天」，若標的價格量級差異大（如比特幣 vs. 低價股），則先將價格取 log 或除以某個縮放係數再套用。
3. **詢問使用者**：若使用者沒有指定，可以主動說明你選擇了哪一種假設（並簡短講清楚為什麼），讓使用者可以自行調整，而不是自己悄悄假設一個刻度就直接給結論。

## 操作建議

1. 先確認樞紐點（高點或低點）與其日期。
2. 詢問或自行合理假設刻度單位，並在回答中清楚寫出你用的假設（例如：「假設 1x1 線 = 每個交易日 X 元」）。
3. 計算至少 1x1、2x1、1x2 三條線的價位供參考；如果使用者要完整扇形，補上其餘角度。
4. 若要視覺化，可用 Visualizer 畫出股價走勢圖並疊加角度扇線（呼叫 `visualize:read_me` 的 `chart` 模組）。
